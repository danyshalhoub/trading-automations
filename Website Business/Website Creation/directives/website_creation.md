# Website Creation System

## Objective

Take the CSV output from the Lead Generation tool and automatically generate a complete, high-converting landing page for each business. Output is a standalone folder per business (HTML + CSS) ready to hand off to the client or deploy immediately.

---

## Setup

### 1. No dependencies required

The script uses only Python standard library modules. No `pip install` needed.

### 2. Run

```bash
python create_websites.py path/to/leads.csv
```

Or run without arguments and enter the CSV path when prompted:

```bash
python create_websites.py
```

---

## Input Format

The script accepts CSVs from both versions of the lead generation tool:

**Current format** (from `lead_gen_apify.py`):

| Column | Description |
|--------|-------------|
| `Business Name` | Business name from Google Maps |
| `Phone` | Phone number (or `N/A`) |
| `Category` | Primary Google Maps category |
| `Location` | Full address string (or `N/A`) |
| `Services` | Comma-separated category list |

**Legacy format** (older CSV exports):

`Company Name`, `Phone Number`, `What the Business Sells`, `Address`

The script resolves both automatically — no reformatting needed. Old CSVs that include `Main Colors` columns are still accepted; values are used if valid hex codes are present.

---

## Output Structure

```
Website Creation/outputs/
└── {business_slug}/
    ├── index.html      ← complete landing page
    └── style.css       ← all styles, color-themed per business
```

Each folder is a self-contained website. To deploy: zip the folder and upload to any web host, or drag into a service like Netlify Drop.

---

## How Industry Detection Works

The script reads the `Category` and `Services` fields and searches for industry keywords. The first match wins:

| Keyword | Industry | Default CTA |
|---------|----------|-------------|
| `plumb` | Plumbing | Call Now |
| `heat` | Heating / HVAC | Call Now |
| `air condition` | Air Conditioning | Call Now |
| `roof` | Roofing | Get Free Estimate |
| `landscap` | Landscaping | Get Free Quote |
| `spa` | Med Spa / Wellness | Book Consultation |
| `salon` | Hair / Beauty Salon | Book Appointment |
| `restaurant` | Restaurant / Food | View Menu |
| *(no match)* | Default / General | Contact Us |

Each industry has its own: headline template, color scheme, trust tagline, about intro, service icon defaults, CTA button label, and highlights strip text.

---

## Color Logic

Industry default colors are used for all sites. If an old CSV contains valid hex codes in a `Main Colors` column, the first hex becomes the primary brand color and the second (if present) becomes the accent.

---

## Visual Design Features

Each generated site includes:

- **Typography**: Barlow (bold sans-serif) for headlines + Inter for body text. Headlines use font-weight 700-800 for authority.
- **Hero section**: Full-width gradient, bold headline, tagline, large clickable phone number (when available), rectangular CTA buttons
- **Services grid**: White cards with primary-color bottom accent border, icon + title + description
- **Trust bar**: Four generic trust points on a solid primary-color band (one strip only — no duplicate section)
- **About section**: Two-column layout — left side has business intro paragraph and 3 industry-specific bullet points; right side has a contact card with business name, address, and phone
- **Contact CTA**: Colored primary-background section with white text and white button. Shows phone button or contact form if no phone.
- **Footer**: Dark (`#0D1117`) with business name, city, and nav links

---

## Writing and Style Rules

These rules apply to all copy generated by the system:

- **No emojis.** Emojis are informal and undermine the professional tone of trade and service businesses.
- **No em dashes.** Never use `—` in any generated text. Use a period or the word "and" instead.
- **Minimal copy.** Every section should say what it needs to say and nothing more. Do not pad with filler phrases.

---

## What Is Never Generated

This is critical. The script will **never** produce:

- Customer reviews or testimonials
- Years in business or founding date
- Certifications or licenses (only "Licensed & Insured" in the trust bar, which every local business can claim)
- Awards or recognition
- Specific guarantees (money-back, satisfaction, etc.)
- Client counts or project counts
- Pricing

Any fabricated specificity destroys trust if the client or their customers notice. When in doubt, keep it broad.

---

## Graceful Degradation

| Missing Field | Behavior |
|---|---|
| `business_name` = N/A | Uses "Local Business" as fallback name |
| `phone` = N/A | Phone links removed; contact form shown in final CTA |
| `location` = N/A | City becomes "Your Area"; address omitted from About section |
| `services` = N/A | 3 generic industry-appropriate service cards used |
| `category` = N/A | Industry detection falls back to services string, then default |

The script processes each row independently — one failed row never stops the rest of the batch.

---

## Post-Run Quality Check

After running, spot-check a few sites before delivering:

1. **Open in browser** — does the page load without errors?
2. **Hero** — is the dot-grid pattern visible? Does the location badge show the correct city?
3. **Phone links** — click the CTA button; does it dial?
4. **Mobile view** — resize to ~375px; does the hamburger menu work? Are buttons full-width?
5. **Colors** — is the primary color readable? Enough contrast on the hero?
6. **Services section** — do all cards show real content (not placeholder text)?
7. **Footer** — correct business name and city?

## Extending the Tool

### Add a new industry

In `create_websites.py`, add a new entry to `INDUSTRY_CONFIG` before `"__default__"`:

```python
"keyword": {
    "hero_headline_template": "...",
    "cta_primary": "...",
    "cta_icon": "fa-...",
    "trust_tagline": "...",
    "about_intro": "...",
    "colors": {"primary": "#...", "accent": "#...", "bg": "#...", "text": "#..."},
    "service_icon_default": "fa-...",
    "highlights": ["Point One", "Point Two", "Point Three"],
},
```

Add matching fallback cards to `INDUSTRY_FALLBACK_CARDS` using the same keyword.

### Add a new service definition

In `SERVICE_DEFINITIONS`, add:

```python
"exact google maps category name": {
    "title": "Human-Readable Service Name",
    "desc": "One or two sentence description.",
    "icon": "fa-icon-name",
},
```

The key must be the Google Maps category string in lowercase (e.g. `"plumber"`, `"heating contractor"`).

Font Awesome 6 Free icon names: https://fontawesome.com/search?o=r&m=free

---

## Notes

- **Re-running overwrites existing output** for the same business slug. This is intentional — it allows re-generating after fixing the CSV.
- **The contact form is a placeholder.** The `onsubmit="return false;"` prevents submission. To make it functional, the client needs a backend service (Formspree, Netlify Forms, etc.). Note this when delivering.
- **Slug collisions** are handled automatically — if two businesses produce the same folder name, the second gets `_2` appended.
